概要#

ローカルLLMのPCを組むとき、つい「演算が速いGPU」を探しがちだが、実際の快適さはほぼVRAM容量で決まる。16GBだと70Bは動いても遅く、24GBでようやく実用、32GB以上で余裕が出る、という感覚に近い。

自分の用途は次のとおり。

  • 70Bを快適に回す
  • 将来は235Bクラスまで伸ばしたい
  • AIエージェント（OpenHands、Aider、Claude Code互換など）
  • Dockerを多数、Kubernetesも
  • 長く使える構成にしたい

この前提なら、「最初に安い1枚構成で終わらせる」より、「後から4枚まで足せる土台を先に作る」方が結果的に安く済む。以下では GPUなし案 → 増設1〜3フェーズ の順で書き、最後に 0ベースで揃える部品一覧（ケーブル・電源まわり含む） を載せる。

価格・URLは2026年7月時点のサンプル。在庫と相場は動きやすいので、購入前に各店で再確認すること。

前提：見るべきは演算性能ではなくVRAM#

70Bクラスを動かすとき、ボトルネックになりやすいのは計算速度そのものより、モデル全体をVRAMに載せきれるかどうか。載せきれないと一部がメインメモリへ落ち、GPUとRAMの行き来がボトルネックになる。体感は「動くけど使えない」になりやすい。

構成70Bモデルの感触
RTX 5070 Ti 16GB△ 動くことはある。RAMオフロードが多く、かなり遅い
RTX 4090 24GB○ Q4ならある程度使える
RTX 5090 32GB◎ かなり快適
RTX 4090 ×2◎ 非常に快適
RTX PRO 6000 Blackwell 96GB◎◎ フルロードしやすい

「ある程度使える」の目安は、チャット・コード生成・エージェント実行でだいたい 10〜20トークン/秒。16GB構成だと数トークン/秒まで落ちることも珍しくない。

ここを押さえたうえで、段階を分ける。

GPUなし案#

まずはソフト側の検証用。Ollama や llama.cpp で 7B〜32B を試し、エージェントや RAG のパイプラインを組む段階。

項目目安
CPURyzen 9 9900X など、コア数が多めのもの
メモリ64GB〜128GB
SSD2TB NVMe（モデル置き場は別途足す）
GPUなし（または内蔵GPUのみ）
総額の目安20〜35万円程度

向いていること:

  • 小さめモデルでのチャット・要約
  • エージェント周りの設定やプロンプトの検証
  • Docker / Kubernetes の学習環境

向かないこと:

  • 70Bの実用速度
  • エージェントの長時間運用を「快適」と呼べる水準

正直、ここは長く居座る場所ではない。70Bを本気で使うなら、次のフェーズへ早めに進んだ方がいい。ただし「まず手元で動かす」経験値は取れるので、ゼロからの第一歩としては悪くない。

注意点はひとつ。この段階で普通のコンシューマ向けマザーに寄せすぎると、後から4GPUを載せたくなったときに載せ替えが発生する。GPUなし期間が短く、最終的に235Bまで行くつもりなら、最初からワークステーション向け土台を買った方が無駄が少ない。

GPU増設1フェーズ（5090×2 + 4GPU対応の土台）#

ここが本命。目安はだいたい 180〜250万円（2026年はメモリ・GPU相場が高く、以前より上振れしやすい）。

最初から「4枚載せられる筐体」を作り、GPUは2枚だけ入れる。

GPU#

RTX 5090 ×2（32GB ×2）

これで70Bはかなり快適。OpenHands や Aider、コード生成、RAG、軽めの画像・音声まわりまで現実的になる。

将来の並びはこうなる想定。

CPU#

おすすめはどちらか。

  • AMD Threadripper PRO 7965WX（24コア）
  • AMD Threadripper 7970X（32コア）

理由は単純で、PCIeレーンが多いこと。GPUを4枚載せる、Dockerを大量に動かす、Kubernetesも同居させる、という用途だと Ryzen 9950X 系はレーン不足になりやすい。CPUを最上位にするより、レーンとGPU予算を優先した方が効く。

マザーボード#

4GPU対応のワークステーション向け。条件の目安は次のとおり。

  • PCIe x16 スロット ×4以上
  • PCIe Gen5
  • ECC RDIMM対応
  • 10GbEがあると便利
  • フォームファクタは SSI-EEB（ケース側の対応必須）

このクラスでだいたい 20〜25万円。

メモリ#

最初から 256GB ECC RDIMM（例: 64GB ×4）。

Kubernetes、Docker、vLLM、OpenHands、VS Code Server を同時に回すと、128GBではすぐキツくなる。将来 512GB まで伸ばせる構成を選ぶ。2026年はDDR5が高騰しており、ここが予算の大きな塊になりやすい。

SSD#

用途で分ける。

  • OS用: 2TB NVMe Gen4
  • モデル用: 4TB NVMe Gen4

235Bの量子化版でも数百GBになるので、モデル置き場は厚めの方が管理しやすい。

電源・ケース・冷却#

項目選び方
電源ATX 3.0/3.1・12V-2x6対応の 1500〜1600W Titanium/Platinum。5090×2なら1基、×4なら2基構成か200V前提の大容量
ケースSSI-EEB対応フルタワー。GPU4枚が入ること
冷却Threadripper専用クーラー（空冷 or 360mm級）。ケースファンも多めにして、GPU間の熱だまりを避ける

日本の100V・15Aコンセントでは、大容量電源でも実効出力が落ちる機種がある。5090を複数載せるなら、専用回路や200V環境を最初から見積もった方が安全。

この段階でできること#

  • 70Bモデル
  • OpenHands / Aider / コード生成 / RAG
  • Kubernetes、Docker多数
  • Stable Diffusion、音声認識、軽めの動画生成

「ローカルLLMを仕事道具にする」ラインは、だいたいここで超える。

GPU増設2フェーズ（5090をあと2枚）#

追加するのは GPU 2枚だけ。CPU・メモリ・電源・ケースはそのまま、という建て付けにしたい。

ここで現実味が出てくるもの:

  • 235Bクラス（DeepSeek V3.x、Qwen3 235B など）
  • 超長コンテキスト
  • 複数エージェント同時実行
  • Cloud Code 級のエージェントを24時間近く回す運用

ソフトウェアや運用を変えずに、VRAMだけを足すフェーズ。1フェーズで土台を正しく作っていれば、ここはほぼ「カードを差す」作業になる。ただし電源と排熱は別問題で、4枚化の時点で電源を2基構成にする、または家側の配線を見直す必要が出やすい。

GPU増設3フェーズ（メモリを512GBへ）#

必要になったらメモリだけ増設する。

235Bで巨大コンテキストを扱う、エージェントやコンテナをさらに同居させる、といった場面で効く。GPUを増やした直後に必須、というより「運用が重くなってから」でいい。

ここまで来ると、構成の主役はほぼ「VRAM総量」と「ホスト側の余裕」になる。CPUをさらに上げるより、メモリとストレージの余裕を見た方が体感に効きやすい。

予算を抑えたい分岐（RTX 4090単発）#

70Bを視野に入れつつ、総額を抑えたいなら次の構成もあり。

項目目安
GPU中古 RTX 4090（24GB）
CPURyzen 9 9900X / 9950X
メモリ96GB〜128GB（できれば128GB DDR5）
SSD4TB
電源1200W Gold以上・ATX 3.x
総額55〜90万円程度（中古GPU・メモリ相場次第）

Q4なら70Bはある程度使える。予算を広げられるなら、単発でも RTX 5090 32GB の方が満足度は高い。

ただしこの分岐は、後から4GPU化しようとすると不利になりやすい。Ryzen系はPCIeレーンが足りず、マザー・電源・ケースごと載せ替えになりがち。235BやマルチGPU運用まで行くつもりなら、最初から前節の Threadripper 土台に寄せた方が安い。

自分なら、「まず70Bを試したいだけ」なら4090単発、「エージェントとKubernetesまで長期間やる」なら1フェーズのワークステーション構成を選ぶ。

購入の優先順位#

長く使う前提なら、買う順番はだいたいこうなる。

優先度内容
①Threadripper系 CPU + 4GPU対応マザーボード + SSI-EEBケース
②ECCメモリ 256GB
③1600W級 ATX 3.x 電源 + Threadripper対応クーラー
④RTX 5090 ×2
⑤NVMe（OS 2TB + モデル 4TB）
⑥ケーブル・専用回路・（できれば）UPS
⑦将来 RTX 5090 を2枚追加
⑧必要ならメモリを512GBへ

GPUを先に買って筐体を後回しにすると、スロット不足・電源不足・物理干渉で詰まりやすい。土台 → 電源インフラ → VRAM → 拡張、の順が安全。

4GPU化で詰まりやすい点#

5090を4枚並べるときの本丸は、演算性能より物理サイズ・消費電力・冷却。市販の大型5090は3〜4スロット厚が多く、普通に並べると4枚入らないことがある。

4枚前提なら、最初から次を決めておく。

  • ブロワーファン仕様やワークステーション向けGPUを選ぶ
  • 水冷化を前提にする
  • GPU間隔を確保できるシャーシを選ぶ
  • 電源は2基構成、または200V専用回路を確保する

「後から2枚足す」だけでなく、「足せる物理構成を最初に選ぶ」。ここを雑にすると、カード代より載せ替えの方が高くつく。

まとめ#

ローカルLLM用PCは、CPUの見栄えよりVRAMと拡張余地で決まる。

  • GPUなし: 小さめモデルとソフト検証用。長居しない
  • 増設1: 5090×2と4GPU対応の土台。ここが本丸（0ベースならケーブル・専用回路まで含む）
  • 増設2: 5090をあと2枚。235Bや複数エージェントが現実味を帯びる
  • 増設3: メモリを512GBへ。巨大コンテキストや同居プロセスが増えたら

予算を抑えるなら4090単発も悪くない。ただ自分の用途（70B快適運用、235B、エージェント、Kubernetes）を長く続けるなら、最初から4枚載る土台を作った方が結果的に安い、というのが今の結論。

買い物のときは、GPUの値段だけ見て決めない方がいい。2026年時点ではメモリと電源インフラの方が、計画外の出費になりやすい。
